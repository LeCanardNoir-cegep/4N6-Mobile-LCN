Sécurité : contrôle d'accès

Avant la séance (2h)

Le contrôle d'accès (ici) consiste à s'assurer que seules les utilisateurs autorisés peuvent accéder / lire / modifier les données du système.

Nous allons voir plusieurs exemples de serveurs et identifier des problèmes d'accès de plus en plus subtiles. Familiarisez-vous avec avant le cours:

  • NoToken, juste ID
  • Token mais ne vérifie pas ownership
  • Create mais pas update
Séance

Nous discuterons des différences entre les différents projets.

Nous verrons également comment rédiger une FAILLE, un EXPLOIT et un CORRECTIF sur un des exemples.

  • Vérifier si l'utilisateur est authentifié, si ce n'est pas le cas exception
  • Si c'est une lecture, il faut ensuite vérifier que l'utilisateur est le propriétaire (ou a le droit d'accès) sur l'objet demandé
  • Si c'est une écriture, il faut s'assurer qu'on ne peut pas créer des données dans le compte d'un autre utilisateur

Exercice
