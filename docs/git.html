Consignes Git

Git

Nom de votre compte

Chaque étudiant doit créer un compte pour ses travaux d’école sur GitHub.com. Le nom du compte doit permettre de facilement identifier l’étudiant en incluant ses nom et prénom.

Nom de du dépot pour un travail

Chaque travail devra se trouver dans un dépôt de code source. Le nom du dépôt pour un travail individuel doit respecter le format suivant : 

SessionAnnée-NumeroCours-NomTravail-NomPrenom

Par exemple le nom H19-4203N5-WebCrawler-SmithJohn correspond à :

  • La session d’hiver 2019
  • Dans le cours de 4203N5
  • Pour le travail WebCrawler
  • De JohnSmith

Pour un travail d’équipe le nom devra être selon le format: 

SessionAnnée-NumeroCours-NomTravail-Nom1Nom2...
Contributeurs du repo

Tous les dépôts de code correspondant à des travaux évalués doivent être privés.

Vous devez inclure votre professeur dans les collaborateurs de votre dépôt pour qu’il puisse y avoir accès.

Les seuls autres collaborateurs avec accès sont les membres de l’équipe s’il s’agit d’un travail d’équipe.

Rédaction des commits

Quand

Tu dois créer des « commits » régulièrement dans ton projet. Tu dois respecter les règles suivantes pour tes commits :

  • Le code de ton travail compile
  • S’il existe des tests unitaires dans ton projet, tu les as exécutés. Si certains ne passent pas, tu dois justifier dans la description.

Un commit contient généralement des modifications à un ou plusieurs fichiers qui constituent un progrès dans ton projet. Tu ne dois pas attendre trop longtemps pour faire un commit mais tu ne dois pas non plus faire un commit à chaque fois que tu changes un fichier.

Une indication pratique est de faire un commit dès que tu as gagné au moins un point dans le barème de ton travail.

Types de commits

On te demande d'identifier le type de commit. Tu dois commencer le titre de ton commit avec le type.

  • FCT (Fonctionnalité) : tu as implanté un nouvel aspect dans ton code qui fonctionne et augmente la valeur de ton travail. Un bon indicateur peut-être que tu as un point de plus selon le barème du professeur.
  • PROGRES (Progrès) : tu n'as pas fini la fonctionnalité mais il faut que tu t’arrêtes de travailler. Tu décris l’avancement.
  • BUGFIX (Débogage) : tu as résolu un problème d’exécution de ton application. Tu dois t’assurer d’avoir supprimer le TODO correspondant dans le code. Tu dois décrire le problème identifié et la solution car il pourrait apparaître ailleurs plus tard.
  • NETTOYAGE (Nettoyage) : tu supprimes des éléments dans ton code qui étaient nécessaires au début du développement mais qui ne le sont plus.
  • REFACTOR (Refactor) : tu as changé l’organisation de ton code pour le rendre plus lisible mais le comportement reste le même.

Description du commit

La description de ton commit doit être en français. Nous évaluons la qualité du français. Assure toi que tes descriptions passent le correcteur orthographique de Word par exemple.

Les titres de commits ne sont pas obligatoirement des phrases (ils doivent rester courts). Cependant, la description doit contenir des phrases compréhensibles par une personne extérieure.

Exemple de commit

Tu as modifié 3 fichiers dans ton code qui permettent d'implanter la sauvegarde dans un fichier.

  • titre : "FCT sauvegarde de configuration dans fichier
  • description : "J'ai utilisé la librairie Guava pour simplifier la sauvegarde. Le format de sauvegarde est tel que décrit à l'URL https://exemple.com/format"
